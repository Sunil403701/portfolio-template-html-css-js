Site Report

About what challenges I had faced while making it?

While working on my web development portfolio, I encountered several challenges that tested both my technical and design skills. Initially, structuring my HTML was tricky as I struggled to maintain proper semantic structure, ensuring accessibility and SEO best practices. CSS styling was another hurdle, particularly with achieving consistent layouts across different browsers; flexbox and grid layouts often behaved unexpectedly, causing alignment issues. Making the site responsive was also challenging, especially when elements didn’t scale properly on smaller screens. I faced difficulties adding interactive features using JavaScript, where event listeners sometimes failed to trigger or produced unexpected errors.
Choosing the right design elements such as fonts and color schemes was equally challenging; I wanted a clean yet visually appealing look, and I drew inspiration from websites like Dribbble and Awwwards. Ensuring text readability while maintaining aesthetics required several adjustments. Validating my code using W3C's HTML and CSS validators highlighted issues like missing alt attributes, unclosed tags, and incorrect property values, which required careful corrections. Deploying the portfolio also had its share of setbacks; incorrect file paths caused some images and styles to break post-deployment, which I resolved by revising my project structure. Throughout the process, managing time effectively was difficult as I often spent too long fine-tuning minor design details. Despite these challenges, overcoming these obstacles significantly improved my coding skills and understanding of web development best practices.
